LC126 word ladder II

发表于 2017-12-29

Description

Given two words (start and end), and a dictionary, 
find all shortest transformation sequence(s) from start to end, such that: 
Only one letter can be changed at a time. 
Each intermediate word must exist in the dictionary. For example:

Given:
start = "hit"
end = "cog"
dict = ["hot","dot","dog","lot","log"]

Output:
[["hit","hot","dot","dog","cog"],["hit","hot","lot","log","cog"]]

Solution

backtracking(TLE)

  • 递归地遍历没有visit过且与cur_word相差一个字母的word
    • TLE: wordlist很长时时间复杂度极高 case19 TLE
    • wordlist.size() = n, word.length() = m.
    • worst case (任意两个词语相差一个字符)-> 时间复杂度为 n(n-1) … 1 * m = n! * m

bfs+dfs(优化)

  1. 先bfs建图再dfs遍历，降低时间复杂度，避免无关节点的重复遍历。实际上是一个无向图找最短路径的题。
    • bfs：key-word, val-predecessor of the word
    • dfs：同样distance的word可以被视为一层，path由每一层中选出符合条件的node相连组成的
  2. 除此之外添加的一些优化
    • 优化一: 在一开始把wordlist全部存进set，直接检查set是否包含end word。
    • 优化二: 维持Map 记录每个word到start_word的distance
      • new_word如果已经被探测且与start_word距离更近，则从start_word到new_word必然存在一条已被探索到的最短路径，所以不用把cur_word加入new_word前继节点的list。
    • 优化三: 检查word之间是否只相差一个字符时可以把字符逐位替换为另外的25个字符，然后看set里是否包含这个word
      • 当word.length() << wordlist.size(): 遍历wordlist建图需要的时间从 n 降到 25*m
    • 优化四: 建图时map存的是某个节点的前置节点，并且dfs时从end_word向前回溯

双向bfs+dfs (再优化，400+ -> 100+ms)

  1. 同时从起点和终点出发对图进行探索。并且每次选择后继节点较少的一端进行探索
  2. 实现（只附了建图部分的码)
    • set1, set2分别记录两个方向已经探索过的节点，一个set记录unvisited的节点
    • 在每次遍历过程中，存储符合条件的节点以备下一层遍历
    • 当set1 和set2中的节点有重合时，说明已经找到了通路
    • 为了维持建图时插入节点的方向一致，额外用一个boolean标记方向
